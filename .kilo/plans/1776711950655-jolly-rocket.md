# Anime Sword Web Game - Project Plan

## Overview
- **Type**: Action RPG / Beat 'em up / Souls-like hybrid web game
- **Tech**: Phaser.js (recommended for 2D action games)
- **Platform**: Web + Desktop (Electron)
- **Monetization**: Freemium

## Game Design

### Combat System
- Real-time action combat with combo chains
- Souls-like elements: parrysystemtiming-based attacks, stamina management
- Weapon variety: different sword styles, upgrades

### Progression
- Character leveling XP system
- Skill trees for combat styles
- Unlockable weapons/items
- Metroidvania-style exploration with new abilities

### Story/Theme
- Revenge plot + mystery exploration + action-focused gameplay
- Open world areas to explore
- Boss battles with souls-like difficulty

### Art Style
- Hybrid: mix of pixel art backgrounds, anime-style characters, vector UI
- Dynamic combat animations

## Technical Architecture

### Phase 1: Core Foundation (Weeks 1-2)
- Project setup with Phaser.js
- Basic game loop and rendering
- Input handling (keyboard/mouse)
- Basic collision system

### Phase 2: Combat System (Weeks 3-6)
- Player controller with movement
- Attack combo system
- Parry/timing mechanics
- Enemy AI (basic + advanced)
- Damage and health system

### Phase 3: Content (Weeks 7-10)
- Multiple playable areas
- Character progression system
- Weapons and upgrades
- Basic story mode

### Phase 4: Polish & Release (Weeks 11-12+)
- UI/UX improvements
- Save system
- Desktop build (Electron)
- Performance optimization

## Tech Stack
- **Game Engine**: Phaser.js 3
- **Language**: JavaScript/TypeScript
- **Desktop**: Electron
- **Build**: Webpack/Vite
- **Art**: Aseprite (pixel), external tools for anime style

## MVP Features
1. Player character with movement and attacks
2. Basic enemy types
3. One playable area
4. Simple health/combat UI
5. Web build

## Post-MVP
- More areas and content
- Boss fights
- Weapon variety
- Co-op / PvP modes (future)
